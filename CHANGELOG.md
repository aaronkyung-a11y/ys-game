## v0.29 (2026-07-15) — BGM 추가 (Skybound Caravan)

**음원 처리**
- 원본 6.15MB / 200kbps / 4분 18초 → **96kbps로 재인코딩 (2.9MB, 절반 이하)**. 게임 BGM엔 200kbps가 과함.
- 원본 끝부분이 페이드아웃돼 있어서 그냥 loop하면 소리가 사라졌다 갑자기 커지는 이음새가 생김 (끝 -38.6dB vs 시작 -27.5dB).
  → 페이드아웃 꼬리를 잘라내고(254초) 짧은 페이드인만 유지 → 이음새 -24.9dB / -29.8dB로 정합.

**임베드 방식 (중요)**
- base64로 index.html에 박지 **않음**. 6MB가 base64로 8MB가 되고, v0.27에서 HTML을 network-first로 바꿨기 때문에 **매 로드마다 8MB를 새로 받게 됨** — 모바일에서 치명적.
- 대신 `bgm.mp3` 별도 파일 + 서비스워커 정적 캐시(cache-first)에 등록. 첫 로드 후엔 오프라인에서도 재생되고 재다운로드 없음.

**재생 로직**
- 브라우저 자동재생 정책 때문에 로드 시 `play()`는 실패함 → 첫 탭/키 입력(user gesture)에서 재생 시작.
- 우상단 ♪ 버튼으로 on/off. 설정은 `localStorage`에 저장되어 다음 실행에도 유지.
- 탭/앱이 백그라운드로 가면 자동 일시정지, 돌아오면 재개.
- 볼륨 0.45 기본.

## v0.28 (2026-07-15) — 고블린 스프라이트 + 적 4방향 렌더링

**grunt → 고블린 스프라이트로 교체**
- 신규 4x3 시트 (512x384) 임베드: 열 = down/up/left/right, 행 = walkA/walkB/attack.
- 제미나이 생성분 12칸 전부 방향 정확 (주인공 때와 달리 LEFT까지 맞음). 걷기 2프레임 차이도 4방향 전부 검증 (mean diff 21~26).
- LEFT는 그래도 RIGHT 반전으로 대체 — 일관성 보장.
- 배경은 테두리 flood-fill로만 제거, 발 기준 정렬, 주인공보다 작게(0.80 스케일) 배치.

**적 4방향 렌더링 (신규)**
- 기존: 적은 `facingLeft` 좌우 반전만 있고 상하 개념이 없었음. 원래 스프라이트가 방향 없는 도형이라 넘어갔던 부분.
- 추가: `dir4(vx,vy)` 헬퍼 — 이동 벡터의 우세축으로 4방향 판정.
- 각 적에 `face`/`walkT`/`px,py` 상태 추가. **실제 이동량**으로 방향을 정하고, 멈춰 있으면 플레이어를 바라봄.
- 걷기 애니메이션은 이동 거리에 비례해서 진행 — 속도가 달라도 발이 미끄러지지 않음.
- 러시 중이거나 방금 때린 직후에는 attack 행 사용.
- 검증: 선회하는 고블린에서 down/left/right 방향 전환 확인, walkT 정상 증가.

**남은 것**: charger·shooter는 아직 기존 도형 스프라이트. shooter 투사체는 돌 던지기로 교체 예정.

## v0.27 (2026-07-15) — 서비스워커 캐시 버그 수정 (버전이 안 올라가던 문제)

**증상**: 레포에는 v0.26이 올라가 있는데 브라우저/앱에서는 계속 v0.25가 뜸.

**원인 (내 설계 실수)**
- `sw.js`의 fetch 핸들러가 **cache-first**였음: `caches.match(req).then(hit => hit || fetch(req))`.
- 게임 전체가 `index.html` 한 파일에 들어있는데, 캐시에 index.html이 있으면 네트워크를 아예 조회하지 않음.
- 캐시 이름(`ys-v0_26`)을 올려도 브라우저가 **sw.js 자체를 캐시에서** 받으면 새 워커가 설치되지 않아 영원히 구버전에 잠김.
- 즉 "캐시 이름만 올리면 갱신된다"는 건 틀렸음.

**수정**
- HTML은 **network-first**로 전환: 항상 네트워크를 먼저 시도하고(`cache: 'no-store'`), 실패할 때만 캐시로 폴백. 오프라인 동작은 유지.
- 아이콘·매니페스트 등 정적 자산은 cache-first 유지 (자주 안 바뀜).
- 페이지 쪽에 자동 갱신 로직 추가: 로드 시 + 60초마다 `reg.update()`, 새 워커 감지되면 `SKIP_WAITING` → `controllerchange`에서 1회 리로드.

**결과**: 앞으로는 새로고침만 하면 최신 빌드가 뜸. 앱 삭제/재설치 불필요.

## v0.26 (2026-07-15) — 적 AI 개선

**그런트: 직선 돌진 → 포위 + 러시**
- 기존: `e.x+=dx/d*spd` 한 줄이 전부. 무조건 플레이어를 향해 직선으로 밀고 들어와서, 여러 마리가 한 줄로 늘어서거나 한 덩어리로 뭉쳤음.
- 변경: 거리별 3단계 행동
  - 300px 초과 — 그냥 접근
  - 240~120px — 접근 + 측면 선회(orbit) 혼합. 링을 유지하며 플레이어를 둘러쌈
  - 랜덤 확률로 `rush` 상태 진입 — 1.9배 속도로 직선 돌진 후 재정비(hesitate)
- 개체마다 `orbitDir`(선회 방향), `orbitAmt`(선회 강도), `wobble`(흔들림 위상)이 랜덤 — 같은 종류라도 움직임이 제각각.
- 검증: 5마리 기준 플레이어 주변 **132° 분산**(한쪽 쏠림 없음), 최근접 14px, 데미지 정상 인가.

**적 분리(separation)**
- 기존: 적끼리 겹침 판정이 아예 없어서 같은 지점으로 수렴 → 여러 마리가 한 마리처럼 보였음.
- 추가: 매 프레임 쌍별 겹침 검사 후 밀어냄 (`SEPARATION = 0.55`).
- 돌진 중인 charger는 밀리지 않고 뚫고 지나감 — 돌진의 위압감 유지.
- 검증: 8마리를 같은 좌표에 스폰 → 겹침 14쌍/25px가 1.5초 후 4쌍/3px로 해소.

**개발 노트**
- 첫 구현에서 `inward = d>90 ? 1 : -0.5`로 잡았더니 그런트가 88px 링에서 영원히 맴돌며 공격을 안 함(데미지 0). 회귀 테스트로 잡아내서 `rush` 상태를 추가하고 링 반경을 120px로 조정.

## v0.25 (2026-07-15) — 3프레임 스윙 모션

**공격이 정지 자세 → 스윙 애니메이션으로**
- 기존: 방향당 공격 그림이 1장뿐이라 칼이 한 자세에 고정. 휘두르는 느낌이 없었음.
- 추가: 신규 4x3 스윙 시트 (512x384) 임베드.
  - 열 = 방향 (down / up / left / right)
  - 행 = 스윙 프레임 (windup / strike / follow-through)
- `drawSwing()` 추가. 공격 중에는 기존 `drawHero` 대신 이 시트를 사용.
- 프레임 배분을 스윙 진행도(`1 - atkT/sw.total`)에 매핑:
  - 0~30% = windup (칼 머리 뒤)
  - 30~70% = strike (칼 앞으로 완전히 뻗음) — 데미지 판정 창과 일치시킴
  - 70~100% = follow-through (칼이 몸을 가로질러 아래로)
- 검증: 콤보 1타(16프레임) 기준 0→1→2 순서대로 재생 확인.

**스프라이트 소스**
- Gemini 생성 → 테두리 flood-fill로 배경만 투명화 (옷·피부 색 보존).
- 발 위치 기준으로 정렬해서 프레임 간 캐릭터가 튀지 않게 함.
- LEFT는 RIGHT를 좌우 반전해서 생성 — 방향 일관성 확보.
- `attack_swing_sheet.png`로 레포에도 원본 보관.

## v0.24 (2026-07-15) — 감속 중 걷기 자세 유지

**walk → idle 전환 부드럽게**
- 원인: 상태 선택이 `player._moving`(입력 유지 여부)를 봤음. 키를 떼는 순간 아직 관성으로 미끄러지는 중인데도 즉시 idle 자세로 바뀜 — 발이 멈췄는데 몸이 미끄러지는 부자연스러움.
- 수정: 입력이 아니라 **실제 속도**(`IDLE_SPD = 0.6`)를 기준으로 판정. 감속 중에는 walk 유지, 거의 멈췄을 때만 idle.
- 검증: 키 릴리즈 후 속도 3.9 → 0.88까지 walk 유지, 0.42부터 idle 전환.

**숨쉬기 위상 리셋**
- 이동/공격/대시 중에는 `breath = 0`으로 리셋. 멈출 때 호흡 중간부터 튀어나오지 않고 항상 새 호흡으로 시작.

## v0.23 (2026-07-15) — 대시공격 자세 버그 수정 & 숨쉬기

**대시공격 시 공격 자세 (버그 수정)**
- 원인: 렌더 상태 선택이 `player.dashAtk ? 'dash' : 'attack'` — 대시공격일 때 **대시 자세**를 유지해서 칼을 휘두르는 모션이 아예 나오지 않았음.
- 수정: `if(player.atkT>0) state = 'attack'` — 대시공격도 스윙이므로 attack 행을 쓴다.
- 대시 잔상 틴트(`#bfe4ff`)는 그대로 유지해서 "대시 중 공격"이라는 시각적 구분은 살림.
- 검증: 진짜 대시공격(`dashAtk=true`) 14프레임 전부 attack 포즈 확인.

**Idle 숨쉬기 이펙트**
- 기존: 가만히 서 있으면 완전 정지 — 마네킹처럼 보임.
- 추가: `breath` 위상 상태. idle일 때만 세로 스쿼시/스트레치 (약 2% 흉부 상하) + 미세한 바브.
- 발이 뜨지 않도록 `yOff`로 높이 변화의 절반을 보정.
- HP가 낮을수록 호흡이 빨라짐 (`0.055 + (1-hpFrac)*0.05`) — 지쳐서 헐떡이는 느낌.

## v0.22 (2026-07-15) — 주인공 행동 자연스러움 개선

**이동에 관성 추가**
- 기존: `x += ix*spd` — 속도가 0에서 최대로 즉시 점프. 출발/정지가 이진적.
- 변경: `vx/vy` 속도 상태 추가. 목표 속도로 가속(`ACCEL 0.28`), 입력을 놓으면 감속(`DECEL 0.22`).
- 결과: 출발 시 3.7 → 5.0 램프업, 정지 시 2.4 → 0 활강.

**방향 전환 보간**
- 기존: 입력 즉시 facing 벡터가 스냅 — 로봇처럼 꺾임.
- 변경: `fang`(각도) 상태 추가, 최단 회전 방향으로 프레임당 `TURN_RATE 0.35` 보간.
- 결과: 0 → -1.57 전환이 4프레임에 걸쳐 부드럽게.

**공격 중 전진 (lunge)**
- 기존: 스윙 중 완전 정지 (`atkT>0`이면 이동 코드 자체를 건너뜀).
- 변경: 액티브 프레임 동안 facing 방향으로 전진력 추가 (일반 1.5, 대시공격 3.2). 스윙 중 속도는 0.80씩 감쇠.
- 결과: 칼을 휘두르며 몸이 따라 들어감. 이스 특유의 "몸통 박치기" 느낌에 근접.

**대시 감속 곡선**
- 기존: `DASH_SPD` 고정 속도로 11프레임 슬라이드 — 얼음판 위를 미끄러지는 느낌.
- 변경: `s = DASH_SPD * (0.45 + 0.55*t²)` — 초반 폭발, 후반 감속. 대시 종료 시 잔여 속도 50% 캐리.
- 결과: 프레임별 13.6 → 7.0 테이퍼 (측정치).

**대시 방향 즉시 확정**
- 대시는 turn 보간을 무시하고 현재 입력 방향으로 즉시 스냅. 회전 지연 때문에 원치 않는 방향으로 대시하는 문제 방지.

**테스트**
- Playwright 헤드리스 회귀 테스트로 4개 동작 모두 검증 (가속 램프 / 감속 활강 / 방향 보간 / 대시 테이퍼).
- JS 문법 검사 통과, 런타임 에러 없음.

### 튜닝 상수 (index.html)
```
ACCEL     = 0.28   // 높을수록 즉각적
DECEL     = 0.22   // 낮을수록 미끄러짐
TURN_RATE = 0.35   // 1.0 = 즉시 스냅
```
